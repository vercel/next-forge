While next-forge ships with Vitest for testing, tracking flaky tests across a monorepo can be challenging. [Trunk Flaky Tests](https://trunk.io/flaky-tests) automatically detects flaky tests, quarantines them so they don't block CI, and helps you eliminate them over time.

## Overview

Trunk Flaky Tests analyzes your JUnit XML test reports over time to identify tests with inconsistent pass/fail behavior. It provides:

- **Automatic flaky test detection** across all your workspace packages
- **Quarantining** so flaky tests don't block PRs
- **AI-powered failure grouping** to help you fix root causes
- **History tracking** to see test health trends

## Signing up

Create a Trunk account at [app.trunk.io](https://app.trunk.io) and connect your GitHub repository.

## Adding JUnit reporters

Update each Vitest config to output JUnit XML reports. In `apps/app/vitest.config.mts` and `apps/api/vitest.config.mts`, add the JUnit reporter:

```ts title="apps/app/vitest.config.mts"
export default defineConfig({
  // ...existing config
  test: {
    environment: "jsdom",
    reporters: [
      "default",
      ["junit", { outputFile: "./junit.xml", addFileAttribute: true }],
    ],
  },
});
```

<Tip>
  Disable automatic test retries in Vitest, as retries can mask flaky behavior and reduce detection accuracy.
</Tip>

## Uploading results from CI

Add the Trunk analytics uploader step to your test workflow. The step should run after your test command, with `if: always()` so results are uploaded even when tests fail:

```yaml title=".github/workflows/test.yml"
- name: Run tests
  run: bun run test
  continue-on-error: true

- name: Upload test results to Trunk Flaky Tests
  if: always()
  uses: trunk-io/analytics-uploader@v1
  with:
    junit-paths: '**/junit.xml'
    org-slug: $\{{ vars.TRUNK_ORG_SLUG }}
    token: $\{{ secrets.TRUNK_API_TOKEN }}
```

## Required secrets

Add the following to your GitHub repository secrets:

- **`TRUNK_API_TOKEN`** — Your API token from [Trunk organization settings](https://app.trunk.io)
- **`TRUNK_ORG_SLUG`** — Your Trunk organization slug (can be set as a repository variable instead of a secret)

## Benefits

- **Reduced CI friction** — Flaky tests are automatically quarantined so they stop blocking legitimate PRs.
- **Monorepo-aware** — Tracks tests across all workspace packages (`apps/app`, `apps/api`, etc.) in a single dashboard.
- **Zero runtime overhead** — Trunk Flaky Tests only processes JUnit XML reports in CI; it adds nothing to your production bundle.

For more information, visit the [Trunk Flaky Tests documentation](https://docs.trunk.io/flaky-tests).
